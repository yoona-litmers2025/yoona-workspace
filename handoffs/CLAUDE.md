# Handoff Defaults

All output in this directory is **English** — for developers and designers.

## Audience
- Vietnamese developers: need precise technical specs
- Vietnamese designers: need clear UX/UI requirements with context

## Tone
- Direct, no pleasantries needed
- Technical precision over politeness
- If something is ambiguous, say so explicitly

## Quality Checklist
Before finalizing any handoff document, verify:
- [ ] Has clear summary (what and why)
- [ ] Requirements are atomic and testable
- [ ] Acceptance criteria are specific
- [ ] Edge cases are listed
- [ ] Priority is assigned
- [ ] No Korean text remains untranslated (except in "Original" reference sections)
- [ ] All domain terms match the client glossary
- [ ] Ambiguities are flagged, not silently resolved
- [ ] "Translated" vs "Inferred" content is clearly separated
