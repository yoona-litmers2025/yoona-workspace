# Client Communication Defaults

All output in this directory is **Korean** unless explicitly requested otherwise.

## Tone
- 기본: 해요체 (메시지, 채팅)
- 공식 문서: 합쇼체
- 따뜻하지만 프로페셔널한 톤 유지
- 이모지 사용 금지 (클라이언트가 요청하지 않는 한)

## Structure
- Lead with the most important information
- Keep paragraphs short (2-3 sentences max)
- Use headers and bullet points for readability
- End with clear next steps or questions

## When translating dev updates
- Remove technical jargon unless the client is technical
- Convert estimates from dev time to calendar dates
- Flag risks early with proposed solutions, not just problems
